## 5. Findings — no defects; one reconciliation, one decision, two carry-forwards resolved

### 5.1 — RECONCILE (12c §5.5): the media "% OFF" chip is now consistent with the card price
Prompt-15 §4.6 asks to "keep the honest discount badge on the media consistent with the card price." The 12c §5.5 carry-forward flagged that the chip was driven purely by `comparePrice > price` and ignored `priceType`, so a tiered/onEnquiry product with a `comparePrice` (e.g. id 1: 420 vs 340 = 19%) could show a `% OFF` that contradicts its `From ₹299` card. Fixed in `ProductCard.js` ([:75–77](../src/components/storefront/ProductCard.js)): the chip shows only when the card renders a single exact number (`priceType==="exact" && showExactPrice !== false`). Runtime confirms id 1 shows the Bulk-pricing chip and **no** `% OFF`, while id 2 shows `20% OFF`. Sanctioned reconciliation the prompt explicitly requested.

### 5.2 — DECISION (§4.2/§4.6): the exact CARD stays compact — one clean number, discount on the media chip
§4.2 says exact shows the struck compare + `% off` "unchanged from today", and (only on lg/details) the "You save" line. "Today" for the **card** meant a single clean price (Prompt-12 passed no `comparePrice`), with the discount on the media chip; "today" for the **PDP** meant the full compare/discount/savings. So `compact = product && mode==="card"` ([:225](../src/components/storefront/PriceBlock.js)) suppresses the inline struck compare/savings on the card (kept honest via the media chip) and shows the full set on details and for legacy callers. This satisfies both §4.2 and §4.6 and preserves the Prompt-12 card look; the `unitType` suffix is the only card-price addition.

### 5.3 — CARRY-FORWARD resolved (12b §5.4 / 12c §7.5): the struck compare + priceType now live on the card/PDP price
Prompt-12 deferred the struck MRP + priceType-aware card price to prompt 15. Delivered: the PDP exact price shows the struck compare/`% off`/savings; tiered surfaces show `From ₹X` (card) and the full table (PDP); onEnquiry shows the pill. The media chip is reconciled (5.1). Nothing from those carry-forwards remains open.

### 5.4 — NOTE: `showTieredPricing===false` on a tiered product → the compact "From ₹X" even on details
When a merchant hides the table (`showTieredPricing===false`) the details view falls back to the compact `From ₹X / unit` rather than an empty region ([:139](../src/components/storefront/PriceBlock.js)) — honest (still a real min-tier number) and never a blank price. All seeded tiered products set `showTieredPricing:true`, so the table renders on every current PDP.
